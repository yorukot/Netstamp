## Contract Source

The API explorer reads `docs/public/openapi.json`. That file is generated from the TypeSpec contract in `api/`, then copied into the backend so non-production runtime docs can serve the same schema.

```bash
pnpm generate:openapi
```

## Use The Explorer

Open [`/openapi/`](/openapi/) from the docs nav. The public page renders the generated contract with Scalar and groups operations by the tags defined in TypeSpec.

Use it to verify request bodies, response schemas, and route paths before writing additional guides.

Controller environments expose runtime Scalar API docs at `/api/v1/docs` and the matching schema at `/api/v1/openapi.json`. The runtime page is same-origin with the API and can send a personal Bearer token; the public docs explorer remains a reference surface and does not require cross-origin API access.

## Authenticated Routes

Project API routes accept either a valid session cookie or a scoped personal access token. Account, member, invite, and administrator operations remain session-only. See [API Tokens](/docs/guides/api-tokens/) for the Bearer workflow.

:::warning Do not manually edit `docs/public/openapi.json` for route changes. Regenerate it from TypeSpec so docs, web API types, and runtime schema stay together. :::

## Documentation Updates

When a route changes, update the guide or reference page that explains the user-facing behavior. Keep implementation details in backend code and use docs for intent, workflow, and examples.
