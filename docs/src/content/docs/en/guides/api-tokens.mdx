Personal API tokens authenticate non-browser project API requests. They are owned by a user, limited by scopes, expire at a chosen time, and remain subject to the owner's current project membership and role.

## Create a token

1. Open **Account → API tokens**.
2. Choose **Create token**.
3. Enter a name that identifies the consumer, such as `grafana-read-prod` or `ci-status-page-publisher`.
4. Select only the required scopes.
5. Set an expiration.
6. Copy the plaintext `nst_pat_...` value immediately.

Netstamp stores only a hash and an eight-character hint. The plaintext is never returned again.

## Scopes

| Area         | Read                | Write                |
| ------------ | ------------------- | -------------------- |
| Projects     | `projects:read`     | `projects:write`     |
| Probes       | `probes:read`       | `probes:write`       |
| Checks       | `checks:read`       | `checks:write`       |
| Labels       | `labels:read`       | `labels:write`       |
| Assignments  | `assignments:read`  | —                    |
| Results      | `results:read`      | —                    |
| Alerts       | `alerts:read`       | `alerts:write`       |
| Status pages | `status_pages:read` | `status_pages:write` |

A write operation may also require the matching read or project scope as defined by the OpenAPI operation. Confirm the endpoint security requirements in the [API explorer](/openapi/).

## Use a token

```bash
curl --fail --silent --show-error \
  --header 'Authorization: Bearer nst_pat_replace_me' \
  https://netstamp.example.com/api/v1/projects
```

Prefer a secret manager or protected environment variable:

```bash
curl --fail --silent --show-error \
  --header "Authorization: Bearer $NETSTAMP_TOKEN" \
  "$NETSTAMP_URL/api/v1/projects/$NETSTAMP_PROJECT/results/latest"
```

Avoid command history, source files, CI logs, screenshots, and query strings.

## Permission intersection

A request succeeds only when all are true:

```text
valid token
AND not expired
AND required token scope
AND active token owner
AND current project membership
AND sufficient project role
```

Removing the user from a project or reducing the user's role immediately narrows what the token can do without changing the token itself.

## Rotate and revoke

Netstamp does not reveal or mutate an existing token value. To rotate:

1. create a replacement with the same or narrower scopes;
2. update and test the consumer;
3. revoke the old token;
4. confirm requests with the old value fail.

Revoke a token immediately after suspected exposure. Account deactivation and administrative credential recovery also revoke or invalidate access according to the affected authentication state.
