Netstamp is an open-source, self-hosted network observability platform. A central controller stores configuration and results, while lightweight probes run checks from the networks, regions, and machines that matter to you.

Use Netstamp to answer questions such as:

- Is a service reachable from every site, ISP, lab, or edge location?
- Is latency or packet loss local to one route, one probe, or the destination?
- Which hop changed when a path became unstable?
- Is an HTTP failure caused by DNS, TCP, TLS, time to first byte, status code, or a body assertion?
- Is a TLS certificate approaching expiration?
- Which incidents are active, and where should notifications be delivered?

## What Netstamp includes

| Area                   | Capabilities                                                                                                               |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| Checks                 | Ping, TCP connect, HTTP/HTTPS, and ICMP or UDP traceroute                                                                  |
| Distributed collection | Linux amd64 and arm64 probe agents, label-based assignment, IPv4 and IPv6 capability reporting                             |
| Analysis               | Latest results, time series, latency and loss metrics, HTTP timing and certificate inventory, traceroute runs and topology |
| Alerting               | Metric-threshold rules, trigger duration, incidents, and Webhook, Slack, Discord, Telegram, or Email notifications         |
| Sharing                | Public status pages with groups, status/history/latency/map blocks, incident history, theming, and custom CSS              |
| Collaboration          | Projects, invitations, owner/admin/editor/viewer roles, labels, and scoped personal API tokens                             |
| Administration         | Registration and email settings, SMTP, user and system-admin management, and JSON data import/export                       |
| Automation             | A versioned HTTP API, OpenAPI schema, interactive API explorer, health endpoints, and probe runtime API                    |

## System at a glance

```text
Browser or API client
        |
        v
Netstamp controller  <---- signed heartbeat, assignments, results ----  Probe agents
        |
        +---- PostgreSQL + TimescaleDB
        |
        +---- Webhook / Slack / Discord / Telegram / SMTP
```

The production container serves both the React application and `/api/v1`. The controller owns authentication, projects, assignments, result queries, alert evaluation, notifications, public status pages, and the probe installer. PostgreSQL with TimescaleDB stores application data and time-series measurements.

The public API prefix is `/api/v1`. The controller also publishes its exact OpenAPI document at `/api/v1/openapi.json` and interactive Scalar documentation at `/api/v1/docs`.
