專案會隔離各自的監測設定、量測結果、警示、公開頁面與成員資格。每個探測器、檢查、標籤、事件或狀態頁都只屬於一個專案。

## 建立專案

完成註冊後，初始設定會引導你建立或選擇專案。只要系統政策允許建立專案，也可以隨時從專案切換器建立另一個專案。

需要填寫：

- **名稱**：畫面上顯示的名稱，最多 64 個字元。
- **網址代稱**：由小寫英文字母、數字和連字號組成的穩定識別名稱，最多 64 個字元。

專案網址的格式為 `/projects/<slug>/...`，API 操作也會在相同位置使用這個專案識別名稱。

## 邀請成員

1. 開啟專案中的「**成員**」。
2. 輸入對方的電子郵件地址。
3. 選擇「**管理員**」、「**編輯者**」或「**檢視者**」。
4. 傳送邀請。

即使對方尚未註冊，也可以先寄出電子郵件邀請。受邀者登入後會看到待處理的邀請，並可選擇接受或拒絕。擁有者與管理員可以取消尚未接受的邀請。

## 角色

| 角色   | 適用情境                                     |
| ------ | -------------------------------------------- |
| 擁有者 | 完整控制專案，包括刪除專案及指派管理員       |
| 管理員 | 管理專案設定與成員，但不能執行擁有者專屬操作 |
| 編輯者 | 日常管理探測器、檢查、標籤與警示             |
| 檢視者 | 以唯讀方式存取專案及分析量測結果             |

擁有者可以指派管理員、編輯者與檢視者；管理員可以指派編輯者與檢視者。每個專案都必須保留至少一位擁有者，因此最後一位擁有者無法離開專案，也不能被移除。

各項操作的確切權限請參閱[權限](/zh-TW/docs/use/permissions/)。

## 專案設定

使用專案側邊欄中的「**設定**」即可重新命名專案或變更網址代稱。變更網址代稱會同時改變瀏覽器與 API 使用的網址；請更新仍引用舊識別名稱的書籤、指令碼、狀態頁編輯連結及 API 用戶端。

只有擁有者可以刪除專案。刪除會移除整個專案及其相關設定與資料。確認前請先完成備份，並再次核對目前選擇的專案。

## 系統管理員是不同的角色

只有系統層級的管理員會看到「**系統管理**」頁面。這個角色負責管理整套 Netstamp 系統，並不會自動取得每個專案的資料。若系統管理員也需要存取專案內容，仍須將他加入該專案。
