應用程式的「**總覽**」頁面是目前所選專案的儀表板。你可以在這裡確認探測器群組是否已準備好執行檢查，並前往相關功能進一步釐清或排除問題。

:::note 「總覽」顯示的是目前狀態，不是歷史報告。若要查看一段時間內的量測資料，請前往[分析](/zh-TW/docs/use/insight/)；若要處理事件，請前往[警示與通知](/zh-TW/docs/use/alerts-notifications/)。 :::

## 選擇專案

Netstamp 會將探測器、檢查、標籤、警示規則與狀態頁分別歸入不同專案。解讀總覽資料前，請先使用應用程式導覽列中的專案切換器選擇正確的專案。一個專案的探測器全數正常，並不代表其他專案也處於相同狀態。

如果尚未加入任何專案，請建立專案或接受專案邀請。詳情請參閱[專案與成員](/zh-TW/docs/use/projects-members/)。

## 探測器健康狀態

探測器摘要會將群組分為以下運作狀態：

- **上線**：探測器正持續向控制器傳送心跳訊號。
- **離線**：控制器已超過預期時間沒有收到探測器回報。
- **準備離線**：探測器已停用。它仍保留在專案中，但重新啟用前不會接收一般監測工作。

這些數字包含目前專案的所有探測器。若要查看個別主機的心跳時間、agent 版本、IP 版本支援情形或安裝狀態，請開啟[探測器](/zh-TW/docs/use/probes/)頁面。

## 地圖涵蓋情形

涵蓋情形卡片會比較已註冊的探測器數量，以及其中已設定地理座標的數量。網路地圖只能標示位置已知的探測器，因此地圖上的標記數可能少於探測器總數。

你可以在探測器編輯器中搜尋地址，或手動輸入緯度與經度。地圖上沒有標記，不代表探測器處於離線狀態。

## 已啟用的檢查

檢查摘要會統計已啟用的監測設定，並依類型分組。Netstamp 目前支援：

- Ping；
- TCP 連線；
- HTTP 或 HTTPS；
- Traceroute。

前往[檢查](/zh-TW/docs/use/checks/)即可新增、編輯、停用或指派檢查。只有檢查設定並不代表它一定會執行；選擇器必須至少符合一個可用的探測器。請參閱[標籤與指派](/zh-TW/docs/use/labels-assignments/)確認兩者的關係。

## 網路地圖

地圖會顯示目前專案中已設定位置的探測器。你可以用它找出缺少探測器的地區，並確認監測工作確實從預期的位置執行。

## 探測器清單

清單會顯示近期的一部分探測器，以及各自的狀態與識別資訊。它方便你從整體數字快速前往特定探測器，但不是完整清冊。若要搜尋、篩選、在地圖上查看或管理所有探測器，請開啟「探測器」頁面。

## 實用的問題排查順序

如果總覽顯示異常：

1. 確認目前選擇的是正確專案。
2. 開啟離線或準備離線的探測器，檢查上次心跳時間與安裝狀態。
3. 確認已啟用的檢查會指派到預期的探測器。
4. 使用「分析」比較不同探測器與檢查的近期量測結果。
5. 確認探測器與指派狀態正常後，再檢查事件及通知傳送情形。

依照這個順序，可以分辨問題是來自監測目標、已停用的探測器、長時間未更新的 agent，還是沒有符合任何探測器的選擇器。
