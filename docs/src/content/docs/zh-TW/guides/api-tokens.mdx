## 建立權杖

開啟「**帳號 → API 權杖**」，設定容易辨識的名稱、到期時間，以及整合真正需要的 scope，然後使用目前密碼確認操作。

明文權杖只會顯示一次。關閉對話框前，請將它保存到 secret manager 或受保護的環境變數。Netstamp 只保存加上 key 的 hash，之後無法再次顯示權杖。

## 呼叫 API

將權杖作為 Bearer credential 傳送，並使用 controller 的公開 HTTPS origin 與有版本的 `/api/v1` base path。

```bash
export NETSTAMP_API_TOKEN='nst_pat_...'

curl \
  -H "Authorization: Bearer $NETSTAMP_API_TOKEN" \
  -H "Accept: application/json" \
  https://controller.example.com/api/v1/projects
```

PAT request 不使用瀏覽器 cookie 或 CSRF token。第一版設計給 server-to-server client 使用，不會開放跨來源瀏覽器存取。

## Scope 與專案存取權

權杖 scope 與擁有者目前的專案角色會同時套用。即使權杖具有 `checks:write`，如果擁有者在專案中只有 viewer 角色，仍無法更新 check。從專案移除擁有者或降低角色後，下一個 request 就會套用新的權限。

可用 scope 涵蓋專案、探測器、檢查、標籤、指派、結果、警示與狀態頁面。帳號安全、專案成員、邀請、權杖管理與系統管理仍只能透過 session 操作。

## 到期與輪替

權杖可在 7、30、90 或 365 天後到期。若要零停機輪替，請先建立替代權杖、更新整合並驗證，最後再撤銷舊權杖。

變更密碼、重設密碼、管理員重設密碼或停用帳號，都會撤銷所有個人 API 權杖。登出瀏覽器 session 不會撤銷權杖。

:::warning 絕對不要把權杖放進 URL、前端 bundle、Git repository、聊天訊息或 shell history。若權杖可能外洩，請立即撤銷。 :::

## 錯誤與可觀測性

無效、已到期、已撤銷或屬於停用帳號的 credential 會回傳 `401 AUTH_INVALID_API_TOKEN`。缺少 scope 時回傳 `403 AUTH_INSUFFICIENT_SCOPE`；專案角色不足則維持現有的專案授權錯誤。

Netstamp 會匯出彙總的 PAT 驗證與 request metric，其中不包含權杖或使用者識別資訊。公開 controller 時，請在 reverse proxy 或 API gateway 套用流量限制。
