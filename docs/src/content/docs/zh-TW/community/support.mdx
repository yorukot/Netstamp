Netstamp 是由社群支持的開源專案，不保證回應時間，也不提供商業支援 SLA。

## 尋求協助前

1. 搜尋本站說明文件與 [API 瀏覽器](/zh-TW/openapi/)。
2. 依照[疑難排解](/zh-TW/docs/operate/troubleshooting/)逐步檢查。
3. 搜尋[既有 issue](https://github.com/yorukot/netstamp/issues)。
4. 在安全且可行的情況下，使用最新的相容版本重現問題。
5. 將問題縮小到單一控制器、探測器、檢查或請求路徑。

## 回報錯誤

請使用[錯誤回報範本](https://github.com/yorukot/netstamp/issues/new?template=bug.md)，並提供：

- 受影響的範圍與對使用者造成的影響；
- 完整且可重現的操作步驟；
- 預期行為與實際行為；
- Netstamp 版本、映像檔標籤或 commit；
- 瀏覽器、作業系統、處理器架構與部署方式；
- 時間戳記與時區；
- 已移除敏感資訊的日誌與截圖；
- 其他人可以重複執行的驗證方式。

絕對不要附上密碼、工作階段 cookie、API 權杖、探測器金鑰、SMTP 憑證、Webhook URL、OAuth 密鑰、個人資料或私人監測酬載。

## 提出功能建議

請使用[功能建議範本](https://github.com/yorukot/netstamp/issues/new?template=feature.md)。先說明使用者遇到的問題，再描述希望達成的結果、範圍、不包含的項目與驗收條件；若已知相關程式碼、API、設計或維運限制，也請一併附上。

## 修正說明文件

每一頁都有**編輯此頁**操作。小型修正可以直接送出範圍明確的說明文件 pull request；若產品行為有變更，請同步更新實作與說明文件，避免範例與實際功能逐漸不一致。

## 安全性問題

請勿建立包含漏洞利用方式、機密值或敏感資料的公開 issue。請依照[安全強化](/zh-TW/docs/operate/security/)與程式碼儲存庫的[安全政策](https://github.com/yorukot/netstamp/blob/main/SECURITY.md)回報。

## 支持專案

不寫程式也能協助 Netstamp：

- 在 GitHub 按 Star 並分享[程式碼儲存庫](https://github.com/yorukot/netstamp)；
- 測試新版本，提供可以重現的回饋；
- 改善說明文件、範例、無障礙體驗、翻譯與部署方法；
- 協助檢視 issue 與 pull request；
- 透過 [Ko-fi](https://ko-fi.com/elvismao) 或 [Buy Me a Coffee](https://www.buymeacoffee.com/elvismao)贊助專案。

贊助有助於專案持續開發，但除非另有明確協議，否則不代表能取得優先支援或 SLA。
