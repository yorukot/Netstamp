## Surfaces

`docs/` is the public website and documentation surface. It uses Astro and ships static assets for GitHub Pages. The landing page is rendered as a React island only because it reuses interactive visual components.

`web/` is the service itself. It uses React Router and now redirects `/` to `/login`, keeping product authentication separate from the public site.

`packages/ui/` contains reusable React primitives and shared design tokens. Cross-surface UI like `GlobalFooter` belongs there.

## Request Flow

The backend exposes HTTP routes under `/api/{version}` with chi. The public API contract is authored in TypeSpec, emitted to OpenAPI, and served by the backend for the docs API explorer.

```text
Browser -> web/ React app -> /api/v1/* -> chi -> application service -> PostgreSQL/TimescaleDB
```

:::warning Do not import `web/src` from `docs/`. Shared React code should move into `packages/ui`, while docs-only Astro components should stay in `docs/src`. :::

## Routing

The docs site owns `/`, `/docs/*`, `/openapi/`, and `/storybook/`. The service app owns `/login`, `/register`, and authenticated routes like `/dashboard`, `/probes`, and `/settings`.
