## Workspace Files

The root `package.json`, `pnpm-workspace.yaml`, and `Justfile` define the common entry points. Prefer those commands when a change crosses package boundaries.

```bash
pnpm install
just lint
just test
just build
```

## Docs Settings

The docs app lives in `docs/`. Astro configuration is in `docs/astro.config.mjs`, while shared docs styles live in `docs/src/styles/docs.css`.

Documentation pages are MDX files under `docs/src/content/docs/`. The left sidebar and search index scan those files automatically from their frontmatter.

## Backend And API Assets

Backend source lives in `server/`. The docs API explorer reads `docs/public/openapi.json`, which should be regenerated from backend route definitions when API behavior changes.

## Database Image

Docker Compose uses `TIMESCALEDB_IMAGE` for the PostgreSQL/TimescaleDB service and defaults it to `timescale/timescaledb-ha:pg16`. The default is intentionally heavier because current migrations enable `timescaledb_toolkit` as well as the core `timescaledb` extension.

Goose migrations can enable installed PostgreSQL extensions, but they do not install extension packages into the container image. A smaller non-HA TimescaleDB image is acceptable only if the required schema no longer enables Toolkit, or if the image is customized to include Toolkit. When switching between HA and non-HA TimescaleDB images, update the compose data volume path to match the selected image layout.

## Shared UI

Reusable React primitives live in `packages/ui/` and are consumed by both the service app and docs. Static documentation for those components is published through the Storybook build at `/storybook/`.

## Update Rules

When configuration changes affect local commands, project structure, or documentation behavior, update the closest `AGENTS.md` guide in the same change.
