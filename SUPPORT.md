# Support

Netstamp is a community-supported open-source project. There is no guaranteed response time or commercial support SLA.

Before asking for help, read the [documentation](https://netstamp.dev/docs/), follow the [troubleshooting guide](https://netstamp.dev/docs/operate/troubleshooting/), and search [existing issues](https://github.com/yorukot/netstamp/issues).

- Use the [bug report template](https://github.com/yorukot/netstamp/issues/new?template=bug.md) for reproducible defects.
- Use the [feature request template](https://github.com/yorukot/netstamp/issues/new?template=feature.md) for product proposals.
- Use the [security policy](./SECURITY.md) for suspected vulnerabilities; do not disclose them in a public issue.
- Use a focused pull request for documentation corrections and implementation contributions.

Include the exact version or commit, deployment method, environment, reproduction steps, timestamps with timezone, expected and actual behavior, and sanitized logs. Never include passwords, cookies, personal API tokens, probe secrets, OAuth credentials, SMTP credentials, webhook URLs, or private monitoring payloads.

The complete help, reporting, contribution, and project-funding guidance is in [Help and project support](https://netstamp.dev/docs/community/support/).
